The Shipyard

design → build → release for a fleet of agent-maintained repos

Shipyard installs a crew of five autonomous agents on a repo: a design loop that mines usage signals into proposals, a build loop that takes ratified tickets to PRs, a release loop that critiques diffs cold-context while the code is being written — plus on-call triage and docs. Systemd-timed, event-logged, PR-gated, with a human stamp at every irreversible handoff.

It is the rebuild of guardian-quartet, whose release agent turned out to be re-running the tests the authoring session had already made green — the same signal the author optimized against, a day late — and whose design loop didn’t exist: features started as chat, gated by whatever the human happened to notice. The three-loop model comes from agent-ready-repo.

This page renders the live build data — crew, skills, phase status — from the same file the ticket’s gates require the builder to update. If this page lags reality, a gate is broken.

Three loops, five crew

Two agents sit outside the shipment loops: on-call and docs. Nothing crosses a gate without a human stamp.

▸ Tap a crew member for their skills — each expandable to the real spec: what’s adapted, what’s replaced, what’s new, with sources and decision IDs.

Skills-parity. Agents run the same skill files a human session invokes — polish-ticket, execute-ticket, coverage-audit — installed per-project, one implementation, N callers. The full disposition audit is in each crew’s drawer.

The build

Thirteen phases. 0–8 harden the existing agents under their current names (the ticket’s verified line numbers depend on it); 9–11 rename, unify skills, and add the design loop; 12 judges the whole system. Status below is live data, updated by the builder as a commit gate.

Installable, gated, judged

Per-project install. Canonical role IDs (design, build, release, medic, scribe) live in code and events; display names are a theme:

bash install.sh --project <dir> --theme spacetime   # mentat · helldiver · proctor · suk · chronicler
bash install.sh --project <dir> --theme plain       # design · build · release · medic · scribe

Hardening shipped with it: merge is opt-in (default no), zero CI checks reads as red, the merge→validate→revert loop is tested against real git topologies, every behavior sits behind a test first shown failing.
